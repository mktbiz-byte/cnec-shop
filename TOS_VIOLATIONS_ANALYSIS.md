# YouTube API Terms of Service 위반 분석 및 수정 계획

## 위반 사항 요약

### 1. 프로젝트 번호 통합 (Policy III.D.1c)
- **위반**: API 클라이언트가 하나의 웹사이트에 대해 여러 프로젝트 번호를 사용
- **현재 상황**: 9개의 Google Cloud 프로젝트 사용
- **해결책**: 하나의 프로젝트로 통합 필요

### 2. 인증 토큰 처리 (Policy III.E.4a-g)
- **위반**: 사용자 동의 없이 인증 토큰 저장 및 사용
- **해결책**: 명시적 사용자 동의 시스템 구현 필요

### 3. 독립 메트릭 제공 (Policy III.E.4h)
- **위반**: YouTube API 외부의 독립적으로 계산된 메트릭 제공
- **현재 문제**: "views", "subscribers", "videos" 아이콘에 한국어 텍스트 누락
- **해결책**: 
  - 한국어 용어 추가: "조회수", "구독자", "동영상"
  - 독립 메트릭 제거 또는 YouTube 데이터로만 제한

## 현재 코드 분석

### YouTube API 사용 현황
1. **API 키 관리**: 여러 API 키 로드 밸런싱 시스템 구현
2. **데이터 저장**: 채널 정보를 로컬 데이터베이스에 저장
3. **캐시 시스템**: 24시간 TTL로 캐시 저장
4. **메트릭 표시**: 조회수, 구독자 수, 동영상 수 표시

### 위반 요소
1. **다중 프로젝트**: YOUTUBE_API_KEY_1, YOUTUBE_API_KEY_2 등 여러 키 사용
2. **데이터 저장**: 사용자 동의 없이 채널 데이터 저장
3. **메트릭 텍스트**: 한국어 용어 누락
